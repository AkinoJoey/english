# 0009. 15秒は知覚を測っていなかった —— 混同がディクテーション側で再発した

- 日付: 2026-07-23
- ステータス: 有効
- 関連: [0008](./0008-rehearsal-is-not-perception.md), [0004](./0004-perception-solved-segmentation-is-next.md)

## 文脈

レッスン 08 の一回聴きプロトコル（narrow listening / Matt Pocock / 15〜20秒を1区間、初見で書き取り）の**1本目**を試した報告:

> そもそも15秒程度の文章を覚えるのが大変で、聞き取れていないのか文章を覚えていないか判断ができていない状態でとても難しい。

## 決定

**レッスン 09 の主題を「診断の単位」にした。** 15〜20秒という区間長は shadowing には正しいが、**知覚の診断には長すぎる**と判断し、診断の粒度を「一息の句（≦2秒・7音節）」に変更した。プロトコルは1か所だけ直す（narrow listening も一回聴きも維持）。

## 非自明な洞察

**これが最重要点。**

1. **ユーザーは、このワークショップの中心テーマの2度目の出現を自力で見つけた。** 記録 0008 の核心は「反復 shadowing は<em>記憶からの再生</em>であって知覚ではない」——知覚と記憶の切り分け。今回、**同じ混同がディクテーション側で、逆向きに**起きていた。08 では shadowing が「記憶なのに知覚に見えた」。今回は書き取りが「記憶の失敗なのに知覚の失敗に見える（見分けられない）」。ユーザーは**この混同を、私が指摘する前に症状として言語化した**（「聞けていないのか覚えていないのか判断できない」）。0005〜0008 で伸びてきたメタ認知の、明確な到達点。

2. **プロトコルの根拠コメントが、そもそも間違っていた。** `reference/error-taxonomy.html` の手順1は「15〜20秒の区間を選ぶ / **作業記憶に収まる長さ**」と書いていた。**これが誤り。** 音韻ループが逐語で保持できるのは**約2秒・7音節**まで（[Baddeley et al. 1975, word length effect](https://www.sciencedirect.com/topics/psychology/word-length-effect)）。15秒は40音節超で、器の10倍以上。ユーザーの困難は**設計欠陥の予測どおりの症状**であって、記憶力の問題ではない。パラメトリックな思い込み（「短めの区間なら作業記憶に入る」）を実地報告が反証した。

3. **境界を超える長さは「わざと記憶をテストする」ために作られている。** SLA の elicited imitation 課題は、刺激文を**あえて7音節/2秒より長くする** —— 逐語オウム返しを封じ、意味からの再構成を強制するため（[EI task design](https://www.sciencedirect.com/science/article/pii/S277276612400082X)）。つまり長い区間は**構造的に知覚をテストしない**。ユーザーが測りたいのは真逆（純粋な音の処理）なのに、プロトコルは知覚を測れない長さを指定していた。**目的と手段が逆を向いていた。**

4. **直しは「時計を2つに分ける」。** 同じ15秒の音源を、**知覚プローブ（一息の句・≦2秒・忘れる前に書ける長さ）**と**shadowing（15秒丸ごと・記憶が入って当然）**の2つの粒度で使う。区間長を捨てるのではなく、**目的ごとに単位を変える**。0008 の「道具は捨てず使いどころを限定する」と同じ構え。

5. **句単位にすると、報告の情報密度がさらに上がる。** 「分類できなかった誤り」が、これで**純粋に知覚の穴**に絞られる（「覚えられなかった」が混ざらない）。0008 が導入した「分類ごとの回数」フィルタの精度が上がる。

## 実装したこと

- **`lessons/0009-the-two-second-wall.html` を新規作成。** 音韻ループ（2秒/7音節）→ elicited imitation の反転 → 診断の単位を句に、の3段。練習は**同じ16語文を一息の句3つに割った cold dictation**（各1回聴き）＋「つなげるとこの文になる」という種明かし。新しい部品は作らず、既存 `dictation`（`data-cold`＋短い `data-answer`）を再利用
- **`reference/error-taxonomy.html` の手順3を修正。** 誤っていた「15〜20秒＝作業記憶に収まる長さ」を削除し、**診断の単位＝一息の句（≦2秒）／15秒は shadowing 専用**に書き換え。2秒境界の callout を追加
- 宿題を「プロトコルを1か所だけ直す」に限定。持ち帰りを**句単位の未分類誤り**に変更

## 未検証の前提

- **「一息の句に割る」がユーザーにできるか未確認。** 息継ぎ・意味の切れ目で切る、と書いたが、**境目の判断自体が新しいスキル**かもしれない。レッスン末で「切れ目が分からなければ transcript を1つ持ってきて一緒に割る」と逃げ道を用意した。次回「割れなかった」と返ってきたら、**句境界の取り方を主題にした練習**（intonation phrase の切り出し）が次のレッスン
- **句単位でも一回聴きが厳しい可能性は残る。** その場合は句の中でさらに（内容語だけ→全部）の段階ディクテーション（`data-say`）に落とす
- **合成音声の限界は 0008 のまま。** 句が短いぶん、合成音声が崩しを再現しない影響は相対的に大きくなるかもしれない。実音源が本体という役割分担は維持

## 次の判断ポイント

- 句単位で「純粋な知覚の穴」が分類できて出てきた → **その音素/規則の集中練習**へ。ワークショップが想定してきた通常運転に戻る
- 句に割れなかった / 境目が分からない → **句境界（intonation phrase）の切り出し**を主題に
- 句単位なら一回聴きが楽になった、と実感が返った → **自動性の量（回数）フェーズ**へ本格移行。ミッションの本丸（技術面接での産出、議論への追従）が射程に入る
- 声を変えて合成音声が自然になったかは**依然未確認**（0008 からの持ち越し）。次回聞く

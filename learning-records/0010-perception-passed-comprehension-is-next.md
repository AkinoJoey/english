# 0010. 知覚が通った —— 壁は「実時間の意味構築」に移った

- 日付: 2026-07-23
- ステータス: 有効
- 関連: [0009](./0009-perception-memory-confound-in-dictation.md), [0008](./0008-rehearsal-is-not-perception.md), [0004](./0004-perception-solved-segmentation-is-next.md)

## 文脈

レッスン 09（一息の句に割る）の報告:

> 「一息の句に割る」を意識したらとてもわかりやすくなった。聞き取れなかった場合も分類表で大体分類できている感じ。ただ、聞き取れても文章を理解するスピードが遅く、聞いた内容への理解度が低い感じがする。

「理解が遅い」の中身を絞るため、頭の中の感覚を4択（多選択）で聞いた。回答:

- **「語は分かるが像が遅い」**
- **「後半で前半を忘れる」**

**選ばれなかった**もの: 「日本語に訳している」「背景知識がないと落ちる」。

## 決定

**レッスン 10 の主題を「実時間の意味構築（incremental comprehension）」にした。** 知覚（音 → 語）は達成と判断。09 で予測した「句単位なら楽になった → 自動性フェーズ」の分岐に入った。ただし本丸は自動性の量だけでなく、**意味構築の様式**だった。

## 非自明な洞察

1. **2つの症状は1つの原因だった。** 「像が遅い」と「後半で前半を忘れる」は別問題に見えるが、どちらも**生の『語』を保持しようとして音韻ループ（約2秒）を溢れさせている**ことの表れ。良い聞き手は語を溜めず、句ごとに意味へ畳んで**1つの育つ像**に統合する（immediacy principle / incremental processing、[Nazarieh 2022](https://onlinelibrary.wiley.com/doi/10.1155/2022/3894457), [immediacy+integration](https://sciencedirect.com/science/article/abs/pii/S0079742122000263)）。運ぶのが「語のリスト（増え続ける）」から「1つの像（一定）」に変われば溢れない。

2. **これは L09 の「2秒の壁」が一つ上の層で再発したもの。** 09 は生の**音**が2秒で溢れた → 句に割った。10 は生の**語**が溢れる → 句ごとに意味へ畳む。**同じ構造・同じ直し方**。ワークショップに「溜め込まず、到着順に変換・圧縮する」という反復モチーフが立った。ユーザーに「また同じ壁だ」と示せるのは学習上強い。

3. **診断質問が、翻訳・背景知識という2大容疑者を除外できた。** ユーザーが「日本語に訳している」を**選ばなかった**のは大きい。最も一般的で最も厄介な原因（back-translation 癖）が無いと分かり、レッスンの重心を「翻訳をやめさせる」ではなく「即時統合の癖づけ」に置けた。**選択肢を phenomenology で出し（専門用語を避け）、多選択＋自由記述にした**のは NOTES の方針どおりで、機能した。0008 で懸念した「選択肢が報告を汚染する」を回避できた。

4. **「像が遅い」と「後半で前半を忘れる」で処方が分かれる。** 前者は語→意味変換の**自動化不足** → 薬は時間と量（narrow listening が同一語彙を回すのがそのまま効く、[Hui & Godfroid 2021](https://www.cambridge.org/core/journals/applied-psycholinguistics/article/abs/testing-the-role-of-processing-speed-and-automaticity-in-second-language-listening/E9E7E1F5516B0C6AB2AE1FEAD6E8293D)）。後者は統合の**様式**の問題 → 薬は句ごとの即時統合ドリル。レッスンは「ドリル＝癖」「多聴＝速さ」の両輪として明示した。

5. **英語の語順が即時統合の味方になる。** 英語は動詞・結論を前に置くので、最初の句で像の骨格が立つ。日本語のように動詞を末尾まで待たなくてよい。**到着順のまま像を進めれば遅れは生まれない**。ユーザーは翻訳していないと答えたので「訳をやめろ」ではなく「到着順に像を1手ずつ進めろ」と積極形で書いた。

## 実装したこと

- **`assets/chunk-build.js` を新規作成（再利用可能な部品）。** 一息の句を1つずつ提示し、句ごとに「今わかっている像」を作らせ（retrieval）、`data-gist` で照合、最後に全句を自然速で通し聞き。将来の聴解レッスンで再利用する想定
- `assets/style.css` に `.chunk-build` 系を追加（既存 `--accent-soft` 等のトークンと `.play-btn` の意匠を踏襲）。印刷時は像を常時表示・操作系を非表示
- **`lessons/0010-building-meaning-as-you-go.html` を新規作成。** 2秒の壁の再発 → immediacy → 英語語順の利 → chunk-build ドリル（技術文2本）→ 宿題（分類の後に「像を1手」を足す）
- 宿題を「像がリセットされた場所」を持ち帰る形に。これが統合の切れ目＝次の診断データになる

## 未検証の前提

- **chunk-build 部品は実質未検証。** voice.js 経由で鳴るはずだが、`replaceWith(cloneNode)` でのリスナー除去・通し聞きボタンの差し替えは動作未確認。次回「通しボタンが出たか／鳴ったか」を聞く
- **「像を一言で言う」がユーザーにできるか未確認。** 訳ではなく育ちかけの絵を進める、という感覚が伝わるかは賭け。掴めなければ、像を選択肢で出す（統合の候補を当てさせる）段階を挟む
- **reference への昇格は保留にした。** chunk-build 手順が有効と確認できてから `reference/` に上げる（0008 の「未検証の道具を作り込まない」規律の踏襲）。レッスン末でユーザーに実感を聞き、掴めていれば昇格
- **合成音声の限界は 0008/0009 のまま。** ただし本レッスンは音そのものより意味統合が対象なので、崩しの再現不足の影響は相対的に小さい
- 声を変えて合成音声が自然になったかは**依然未確認**（0008 からの持ち越し）

## 次の判断ポイント

- 像を句ごとに進める感覚が掴めた → **chunk-build を reference に昇格**し、素材を実際の Matt Pocock の transcript に差し替えて継続。ミッションの本丸（技術ミーティングの追従）が射程
- 像がどこでリセットされるか自己診断できない → transcript を一緒に句割り＋像書き出し。統合の切れ目を可視化する回に
- 「像が遅い」だけが残り「忘れる」が消えた → 統合の様式は直った。あとは純粋な量（多聴の比率↑）。レッスンを減らす

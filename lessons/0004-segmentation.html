音を語に切る

今回、あなたは大きく前進しました。それが誤答の中に証拠として残っています。

1. あなたの誤答は、実は成功していた

これを見てください。

正解あなたが書いたもの

これは正解です。 tools are は実際に「トゥーオザー」と鳴ります。tools の dark L が母音化し（前回の内容です）、are が /ər/ に潰れて（前々回の内容です）繋がる。あなたは音を正確に捉えていた。

できなかったのは、その音を語に切ることだけです。

問題の層が変わりました。 レッスン 01〜03 は「音が聞こえるか」の話でした。今日からは「聞こえた音をどこで切るか」の話です。あなたは次の段階に来ています。

他の誤答も、すべて同じ構造です。音は取れていて、境界だけが違う。

正解あなたが聞いた形何が起きたか

4件すべてで、崩れているのは機能語の周辺です。内容語（tools, model, file, packages）はほぼ取れています。

2. あなたは、英語の切り方を持っていない

ここが今日の核心です。そして、これはあなたの努力不足ではありません。

英語の母語話者は、音の流れをこう切っています。

強い音節が来たら、そこが新しい語の先頭。

これを Metrical Segmentation Strategy と呼びます（Cutler & Norris, 1988）。この方略が機能するのは、英語の語の約90%が語頭強勢だからです。強い音節を見つけたら、その手前に切れ目を入れれば、だいたい当たる。

問題は、この方略が日本語には存在しないことです。日本語はモーラ（拍）基準の言語で、すべての拍が等価に刻まれます。強い音節を目印にするという発想自体がありません（Encyclopedia of Language Development）。

だからあなたは、英語を日本語の切り方で切ろうとしている。等間隔に刻んで、全部の音を拾おうとする。すると the model が「デマンド」という等間隔の塊になる。

つまり必要なのは、耳を良くすることではなく、持っていない方略を新しく身につけることです。 これは訓練で獲得できます。今日やるのがそれです。

3. 切る手順

やること
1強い音節だけを拾う。弱い部分は捨ててよい
2強い音節の直前で切る（そこが語頭）
3拾った音節から内容語を復元する
4内容語の間の機能語は、文法から推測する

手順4が、あなたの誤答すべてに効きます。

機能語は聞き取るものではなく、推測するものです。 潰れて鳴っていない以上、耳で捕まえようとするのが間違いです。on a local terminal の a は、聞こえなくて正常です。local terminal という可算名詞句が来た時点で、文法が a を要求する。だから a だと分かる。

あなたが at on local と書いたのは、聞こえた音を全部文字にしようとした結果です。その努力は、この段階では逆効果になります。

4. 練習 — 2段階で切る

今日は同じ音声を2回使います。1段階目は内容語だけ。機能語は無視してください。2段階目で全文を復元します。

(1) まず内容語だけを拾う

機能語（a / of / are / in / the）は5つもありますが、全部無視して正解です。強い音節だけで意味は通ります。

(2) 同じ音声を、今度は全文で

埋め方: couple の前は a、後ろは of（a couple of は固定形）。tools は複数なので are。terminal は特定のものなので the。すべて文法が決めています。

(3) 前回の誤答を、切り方で解き直す

前回 the model を「demand」と融合させた箇所です。強勢は MO に1つだけ。強い音節が1つなら、内容語も1つ。 その手前が語の境界です。

前回の誤答が、今回は解けたはずです。同じ音、同じ速さ —— 変わったのはあなたの切り方だけです。

(4) 実音源から

install は例外です。英語の約10%は語頭強勢ではなく、その多くが接頭辞つきの動詞（in·STALL, de·PLOY, re·TURN, con·NECT）。強い音節の手前に切れ目を入れると stall になってしまうので、接頭辞の分だけ左に戻す必要があります。

the following の the は /ðə/ に潰れ、following の dark L（FOL）と繋がって、あなたには「from パクチー」に聞こえました。強い音節は FOL と PACK の2つ。だから内容語も2つです。

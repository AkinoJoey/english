Lesson 08 · Listening

初見で聴く

「練習すれば重ねられた。でも初見は無理そう」——— そこが今の壁です。名前をつけます。

まず、達成を確認します。 transcript なしで音声に重ねられた —— レッスン 07 の目標は達成です。[ɡoʊɾə] は、もうあなたの口から出る音になりました。
そのうえで、あなたの観察は正確です。 重ねられることと、初見で聞けることは別の能力です。今日はその差が何なのかを扱います。

1. 反復 shadowing は「聞く」を証明しない

15秒を20回繰り返したあと、あなたが音声に重ねられるのはなぜか。候補は2つあります。

入力の役割何が起きているか
知覚音から語を作る音響を毎回、語に変換している
再生きっかけにすぎない記憶から次を出している

20回目の重ねは、ほぼ下の行です。音声は「次はこれ」という合図として働くだけで、語は記憶から出てきます。SLA の実務側でもこれは mindless parroting（理解を伴わない反復）として警戒されています（Conti, 2025）。

※ 正確に言えば、反復 shadowing が新規素材の聴解に転移するかどうかは、私が確認できた範囲では実証されていません。「転移しない」と証明されたわけでもありません。不明です。ただし shadowing が産出（リズム・弱形）を伸ばすことは 44件のレビューで支持されており、レッスン 07 はその目的を果たしました。道具を捨てるのではなく、使いどころを限定します。

2. 初見が難しい理由は、知識ではなく速度

英語の会話速度はおよそ 150 wpm（National Center for Voice and Speech）。技術系の動画はこれより速いことが多い。つまり1語あたり 300〜400 ミリ秒です。

レッスン 01〜06 で渡した規則は、すべて正しく手元にあります。問題は —— 「これは flap かな」と考える時間が、どこにもないことです。

Hui & Godfroid (2021) は、L2 聴解において語彙レベルの処理の速さと安定性が、命題レベルの理解を経由して聴解全体を左右することを示しました。重要なのは、速さだけでなくばらつきの小ささ（変動係数）を自動性の指標として区別した点です（Applied Psycholinguistics 42(5)）。
「速いときも遅いときもある」は、まだ自動化していないということです。

ここから導かれる結論は身も蓋もありません。あなたに今必要なのは、新しい知識ではなく回数です。 ただし、回数の積み方には設計があります。

3. 素材の設計 —— narrow listening

初見素材はふつう、負荷が全方向から来ます。未知語・未知の話題・未知の話者・未知の音。鍛えたいのは音の処理だけなのに、他が邪魔をする。

そこで narrow listening（Krashen 1996）—— 話者と話題を固定し、素材だけを変えます。

同じ話者・同じ領域 × 違う回を4〜5本

同じ語彙と言い回しが何度も回るので、語彙と背景知識の負荷が下がる。残るのは音韻処理だけです。タイ EFL 学習者を対象にした研究では、narrow listening 群は練習していない話題の聴解でも有意に高い伸びを示しました（PASAA, 2022）。

あなたの場合、これはそのままミッションに刺さります。 同じ技術系 YouTuber の動画を5本。語彙は deploy / endpoint / cache / repo と回り続け、話し方の癖も一定。実際の職場の英語も、まさにこの条件（同僚は毎日同じ人、話題は毎日同じ領域）です。

1本を20回より、似た5本を4回ずつ。 1本の反復は暗記に化けますが、5本には化ける先がありません。

4. 一回聴きの規律

実世界に巻き戻しボタンはありません。ミーティングで聞き返せるのはせいぜい1回です。練習も条件を寄せます。

  1. 15〜20秒を選ぶ。transcript はまだ見ない
  2. 1回だけ再生して、書けるだけ書く
  3. もう1回だけ再生して、空白を埋める
  4. transcript と照合する
  5. 誤りを誤り分類表で分類する
  6. その区間を shadowing する（ここで初めて口を使う）

速度は絶対に落としません。 ゆっくり再生は理解を助けますが、それは鍛えている能力ではありません。2回で取れなければ、それは「今日の穴」として記録するだけでよく、埋めるのは transcript の仕事です。

そして手順5が今日の本体です。誤りに名前をつけると、既習の規則を能動的に引き出すことになります —— 読み返すより retrieval のほうが定着します。分類できた誤りは既知の穴。分類できない誤りだけが新しい情報です。

5. 練習 —— 初見、1回聴き

下はゆっくり再生がありません。再生回数が表示されます。既習の規則がばらばらに混ざっているので、どれが来るか予告しません。実際の聴取と同じ条件です。

先に声を選んでください。 読み上げが「ロボットみたいで不自然」との報告を受けて、声を差し替えられるようにしました。これは見た目の問題ではありません —— 機械的な声は丁寧に読み上げるため、flap や弱形といった、まさにこの教材が扱っている現象を作らないことがあります。
下のリストからいくつか試して、一番人間らしいものを選んでください。Google や Siri を含む名前、「ネットワーク」表示のものが自然な傾向です。選択は保存され、全レッスンに反映されます。

まず1回だけ押して、書けるだけ書いてください。

分類は弱形（should be able to）とラ音（a lot of it → [əlɑɾəvɪt]）。立つのは CACHE / LOT だけです。

分類はdark L。all / calls / fall / old の4連発です。ウ段や長母音に聞こえたら、L を疑う。

分類は枠＋ラ音。What it does is it は助走路で、what it は [wʌɾɪt]。あなたが「リターンズ」と聞いた構造そのものです。枠のリファレンス

6. 今週やること

一次資料:
Narrow Listening as a Method to Improve EFL Learners' Listening Comprehension（PASAA, 2022 / 無料）。手順と結果が短くまとまっています。練習していない話題にも伸びが出た点に注目してください。

実地練習 —— 素材を「揃える」ことから始めます。

手順0: 話者を1人に固定する

narrow listening の要は「今後しばらく、この人だけを聴く」と決めることです。毎回違う人を聴くと、声・訛り・速さ・口癖が毎回リセットされ、そのぶんの負荷が音の処理を圧迫します。逆に1人に固定すると、数本目からその人の癖が予測できるようになり、余った処理能力が音に回ります。

推奨: Matt Pocock をそのまま続けてください。 あなたがレッスン 07 で使った動画の話者です。すでに数十回聴いて声に慣れている —— これは narrow listening の観点では、ゼロから始めるより明確に有利です。イギリス英語で、TypeScript という領域も一定しています。

変えたい場合の候補（いずれも transcript あり）:

話者訛り特徴
Matt Pocock英説明が構造化されていて、速度は中程度。継続なら第一候補
Theo (t3.gg)米ライブ配信中心で自然な話し言葉。速く、フィラーも多い。難度は高い
Web Dev Simplified米明瞭でゆっくりめ。最も易しい。厳しすぎると感じたらここへ

「聞き取りやすいか」より「今後も聴きたいか」で選んでください。 数十時間つき合う相手です。

手順1〜4

  1. その人の動画を4〜5本、transcript が取れるものでキューに入れる（YouTube なら概要欄下の「文字起こしを表示」）
  2. 各本から15〜20秒を1区間ずつ選び、上の一回聴きプロトコル（1〜6）を回す
  3. 誤りを分類表で振り分ける
  4. 1本を極めないこと。浅く、5本に散らす

持ってきてほしいもの:

  1. 分類できなかった誤りを1つ。前後1文つきで。分類できたものは報告不要です —— それは既知の穴で、回数で埋まります
  2. 各分類が何回出たかのざっくりした数（弱形2回、dark L 1回…程度で十分）。偏りが見えたら、そこを次のレッスンにします

7. 合成音声の限界について

声を差し替えても、合成音声には原理的な限界があります。読み上げエンジンは基本的に「明瞭に読む」ことを目的に作られており、人間の話者がやる崩し —— go to を [ɡoʊɾə] にする、should be able to を全部 [ə] にする —— を、いつも再現するとは限りません。

したがって、このワークショップの練習には役割分担があります。

役割
レッスン内の音声規則の確認。何が起きるはずかを、短い文で明示的に見る
実音源（narrow listening）訓練の本体。 本物の崩しは、本物の話者からしか来ない

週9〜11時間のうち、レッスンは短く、大半を実音源にというミッションの配分は、この意味でも正しい方針です。上の練習で違和感があったら、合成音声側を疑ってください —— そして、それを報告してください。
